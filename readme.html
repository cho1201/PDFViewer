Google Cloud Console 설정 안내

이 PDF 뷰어 웹 애플리케이션을 사용하려면 Google Cloud Console에서 두 가지 핵심 사용자 인증 정보(API Key, OAuth 2.0 Client ID)를 발급받아야 합니다. 아래 절차를 따라주세요.

1단계: 프로젝트 생성 및 API 사용 설정

  1. Google Cloud Console에 접속하여 새 프로젝트를 생성합니다.
  2. 왼쪽 메뉴에서 [API 및 서비스] > [라이브러리]로 이동합니다.
  3. 'Google Drive API'를 검색하여 [사용] 버튼을 눌러 활성화합니다.

2단계: OAuth 동의 화면 구성

API를 사용하기 전, 사용자에게 보여줄 동의 화면 정보를 설정해야 합니다. (최신 'Google 인증 플랫폼' UI 기준)

  1. [대상(Audience)]: [외부(External)]를 선택하고, '테스트 사용자' 섹션에 본인의 Google 계정 이메일을 반드시 추가합니다.
  2. [브랜딩(Branding)]: 앱 이름(예: `My PDF Viewer`)과 사용자 지원 이메일을 입력합니다.
  3. [데이터 액세스(Data Access)]: [범위 추가 또는 삭제]를 클릭합니다.
    • .../auth/drive.readonly (필수: 파일 읽기)
    • .../auth/drive.file (필수: 메모 저장을 위해 앱이 생성한 파일에만 접근)

    위 두 가지 범위를 검색하여 모두 체크하고 업데이트합니다. '테스트 사용자'로 등록되어 있다면 앱 검증 없이 진행 가능합니다.

3단계: 사용자 인증 정보 생성

이제 로그인 화면에 입력할 두 개의 키를 생성합니다.

A. API 키 생성

  1. [API 및 서비스] > [사용자 인증 정보]로 이동합니다.
  2. [+ 사용자 인증 정보 만들기] > [API 키]를 선택합니다.
  3. 생성된 API 키를 복사하여 보관합니다. (보안을 위해 '애플리케이션 제한사항'에서 'HTTP 리퍼러'를 설정하는 것이 좋지만, 로컬 테스트 중에는 '없음'으로 두어도 됩니다.)

B. OAuth 2.0 클라이언트 ID 생성

  1. [+ 사용자 인증 정보 만들기] > [OAuth 클라이언트 ID]를 선택합니다.
  2. 애플리케이션 유형: 웹 애플리케이션을 선택합니다.
  3. 승인된 자바스크립트 원본 (매우 중요!):

    이 웹 앱을 실행할 주소를 정확히 입력해야 합니다. 로컬 서버로 테스트하는 경우, 주소창에 보이는 주소를 추가하세요.

    • 예: http://localhost:8000
    • 예: http://127.0.0.1:5500
    • 예: https://{본인아이디}.github.io
  4. [만들기]를 클릭하고, 생성된 클라이언트 ID를 복사하여 보관합니다.

4단계: 앱 실행

발급받은 API Key와 Client ID를 PDF 뷰어 페이지의 로그인 화면에 입력하면 모든 기능이 정상적으로 작동합니다.
